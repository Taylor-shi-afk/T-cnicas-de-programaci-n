1. Programación Tradicional:

Estructura: Se basa en una secuencia lógica de pasos y funciones independientes.

Ventaja: Es más directa y fácil de implementar para tareas pequeñas o cálculos simples como este.

Limitación: El flujo de datos es abierto, lo que puede dificultar el mantenimiento si el programa crece demasiado.

2. Programación Orientada a Objetos (POO):

Estructura: Organiza el código en "objetos" que agrupan tanto los datos (atributos) como los comportamientos (métodos).

Ventaja: El encapsulamiento protege los datos internos (como la lista de temperaturas), y la herencia permite extender funcionalidades de forma limpia. Es ideal para sistemas complejos y escalables.

Reflexión: Mientras que la programación tradicional es funcional, la POO permite modelar el problema de una forma más cercana a la realidad, facilitando la reutilización de código.
